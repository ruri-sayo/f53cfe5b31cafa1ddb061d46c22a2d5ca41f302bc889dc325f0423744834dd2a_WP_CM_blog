# ブログ執筆スキル (blog-author)

このスキルは、本ブログの執筆における「標準規格」を定義したものです。新しい記事の作成、既存記事の編集、記事案からの清書を行う際に必ず適用してください。

## 1. フロントマター規格 (Frontmatter)

すべてのMarkdownファイルは、以下の形式のYAMLフロントマターを持つ必要があります。

```yaml
---
title: 記事のタイトル
date:   YYYY-MM-DD HH:MM:SS +0900
categories: [メインカテゴリ, サブカテゴリ]
description: 記事の要約（外部向け、100文字程度）
---
```

### カテゴリの指針
以下の既存カテゴリから適切なものを選択してください。必要に応じて新しいカテゴリを作成しても構いませんが、一貫性を保ってください。
- `Knowledge`: 知識共有、解説記事
- `Essay`: コラム、考察
- `Notice`: お知らせ

## 2. 執筆スタイル (Writing Style)

### 口調とトーン
- **基本口調**: 親しみやすく、かつ知的な「技術者同士のペアプログラミング」のような距離感。
- **語尾**: 原則として「〜です」「〜ます」調（デス・マス調）。ただし、Essayカテゴリなどでは「〜である」調も許容されますが、ユーザーからの指定がない限りは「〜です」「〜ます」を基本とします。
- **視点**: 読者に寄り添いつつ、自信を持って情報を提示する。

### 文章構成
- **見出しレベル**: `##` (H2) を章の切り分け、`###` (H3) を細かな項目に使用します。`#` (H1) は記事タイトルとして扱われるため、本文内では避け、フロントマターの `title` に委ねます。
- **強調**: 重要な用語や結論は `**太字**` で強調します。
- **リスト**: 箇条書きを積極的に使い、可読性を高めます。
- **表**: スペック比較や価格比較などはMarkdownのテーブル形式を使用します。

### 初心者への配慮
- 初心者が躓きそうなポイントには `> ⚠️ **注意:**` や `> 💡 **Tips:**` といった引用ブロックを用いた注釈を入れます。
- 専門用語を最初に出す際は、必要に応じて簡単な補足（パーレン内など）を行います。

## 3. 禁止事項・注意事項
- **曖昧な表現の回避**: 「〜かもしれません」「〜だと思います」を多用せず、事実や推奨事項を明確に述べます。
- **macOSに関する記述**: 自作PC関連の記事では、macOSが原則として自作PCで利用不可（ライセンス外）であることを、ユーザーの意向に沿って「自由の敵」や「囲い込み」といったニュアンスを含めて明確に記述する場合があります（※第一部記事参照）。
- **プレースホルダーの禁止**: 「あとで書く」などのプレースホルダーを残したままにせず、執筆時に可能な限り埋めるか、明確なTODOコメントを残します。

## 4. ワークフロー
記事案（`dev/記事案/*.md`）を `src/content/posts/*.md` に移行する際は、以下のステップを自動的に実行します。
1. 現在の日時を取得し、`date` を設定する。
2. 適切な `categories` と `description` を生成・付与する。
3. 本文の口調が本スキルに沿っているかセルフチェックし、必要に応じて修正する。
